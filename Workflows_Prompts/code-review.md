# 任务：代码审查

## 审查维度
1.  **正确性**：逻辑、边界、输入验证、错误处理
2.  **安全性**：数据泄露、资源管理、依赖
3.  **代码质量**：可读性、命名、重复、规范
4.  **性能**：算法、查询、循环、缓存
5.  **可维护性**：模块化、配置、测试
6.  **最佳实践**：语言特性、API设计

## 输出格式
- **🔴 严重**：[高] 问题+位置+建议
- **🟡 改进**：[中] 优化+理由（+示例）
- **🔵 风险**：[低] 风险+预防
- **📁 文件**：变更对文件整体影响

## 说明
- 审查**git diff**中的整个文件（非仅变更行）
- 反馈保持简洁、可执行
